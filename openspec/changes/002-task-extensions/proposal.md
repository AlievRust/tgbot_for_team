# Proposal: Расширение задач — ответственный, дедлайн, статус, автопросрочка

## Описание изменения
Расширение таблицы задач новыми полями: ответственный (assignee), дедлайн (deadline), статус (status).
Добавление команды /edit для изменения существующей задачи через inline-кнопки.
Автоматическая простановка статуса «просрочено» при наступлении дедлайна с уведомлением в чат.

## Мотивация
Задачам не хватает атрибутов для реального управления: кто отвечает, когда нужно сдать,
какой текущий статус. Без этого список задач — просто памятка, а не инструмент.

## Область применения
- Только групповые чаты (как и весь функционал бота)
- Дедлайн — только дата (без времени), формат YYYY-MM-DD
- Статус «просрочено» выставляется автоматически, вручную — нельзя
- Ответственным может быть только участник из числа известных members группы

## Новые поля в таблице tasks
| Поле | Тип | Default | Описание |
|------|-----|---------|----------|
| assignee | TEXT | NULL | Username ответственного (NULL — не назначен) |
| deadline | TEXT | NULL | Дата дедлайна YYYY-MM-DD (NULL — не задан) |
| status | TEXT | 'в работе' | 'в работе', 'выполнено', 'просрочено' |

## Новая таблица chat_members
| Поле | Тип | Описание |
|------|-----|----------|
| chat_id | TEXT NOT NULL | ID группового чата |
| username | TEXT NOT NULL | Username участника |
| display_name | TEXT NOT NULL | Display name для отображения |
| last_seen | TEXT NOT NULL | Дата последнего взаимодействия |
| PRIMARY KEY | (chat_id, username) | Уникальность пары чат+юзер |

> **Почему отдельная таблица:** Telegram Bot API не позволяет получить список всех участников группы.
> Бот собирает список автоматически: при любом взаимодействии пользователя с ботом в группе
> (команда, ответ на FSM, выбор кнопки) — записывает/обновляет его в chat_members.

## Команда /edit
Поток через inline-кнопки (закрытый список вариантов):

1. `/edit` → бот просит ввести ID задачи
2. Пользователь вводит ID → бот проверяет существование → показывает кнопки:
   - 📌 Назначить ответственного
   - 📅 Установить дедлайн
   - ✅ Сменить статус
3. **Назначить ответственного** → кнопки со списком известных members группы + «❌ Снять»
4. **Установить дедлайн** → кнопки: «Сегодня», «Завтра», «+3 дня», «+7 дней», «❌ Снять»
   (бот рассчитывает дату в GMT+3, показывает в кнопке конкретную дату)
5. **Сменить статус** → кнопки: «В работе», «Выполнено» (без «Просрочено» — только автоматом)

## Автопросрочка
- Фоновая корутина (нативный asyncio, без доп. зависимостей)
- Интервал проверки: раз в 1 час
- Логика: SELECT задач со status='в работе' И deadline не NULL И deadline < сегодня
- Для каждой найденной: UPDATE status='просрочено', отправить сообщение в чат:
  `⚠️ Задача #<id> просрочена! @<assignee> — дедлайн был <deadline>`
  (если assignee NULL — без тега)
- Для определения chat_id задачи нужна привязка задачи к чату (см. дизайн)

## Ограничения
- Дедлайн — только дата, без времени суток
- Статус «просрочено» нельзя установить вручную
- Снять дедлайн/ответственного можно через опцию «❌ Снять» в меню
- Если в группе ещё никто не взаимодействовал с ботом — список members пуст, назначение ответственного невозможно (бот сообщит об этом)
